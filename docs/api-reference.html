MockStar REST API

Turns

/sessions/{id}
POST /sessions/{session_id}/turns Submit an answer (audio + video)
Request — multipart/form-data
FieldTypeRequired
audiofile (audio/webm)Yes
question_textstring (form field)Yes
videofile (video/webm)No
Response 200
FieldType
turn_idstring (UUID)
transcriptstring (empty — populated async)
Non-blocking. Response is immediate. Transcription (Whisper) and CV analysis (OpenCV) run as background tasks and update the in-memory session store. Audio and video bytes are never persisted to disk or database.
POST /sessions/{session_id}/next-question Generate a contextual follow-up question
Response 200
FieldType
questionstring
Gemini 2.5 Flash receives the full conversation history to generate a contextually relevant follow-up. No request body required.
POST /sessions/{session_id}/end Finalise interview and generate feedback
Response 200 — FeedbackResponse
FieldType
feedbackstring (markdown)
scores.answer_relevanceinteger (1–10)
scores.experience_articulationinteger (1–10)
scores.industry_fitinteger (1–10)
scores.clarity_and_structureinteger (1–10)
scores.filler_word_usageinteger (1–10)
scores.eye_contact_and_presenceinteger (1–10) nullable
scores.overallfloat
confidence_scorefloat (0–100) nullable
confidence_labelstring (High/Medium/Low) nullable
observationsstring[] nullable
Waits up to 30 s for pending background tasks to complete, then aggregates CV results, calls Gemini, and writes the full session to Cloud SQL in a single commit. Deletes the session from the in-memory store.

Users

/users/{user_id}
POST /users/{user_id}/resume Upload or replace resume (PDF / DOCX)
Request — multipart/form-data
FieldType
filefile (PDF or DOCX)
Response 200
FieldType
message"ok"
char_countinteger
Upserts to user_resumes — one resume per user. Replaces any existing resume on re-upload.
GET /users/{user_id}/resume Retrieve stored resume text
Response 200
FieldType
resume_textstring
filenamestring nullable
uploaded_atstring (ISO 8601)
Returns 404 if no resume is on file. Used by the dashboard to gate access to onboarding.
POST /users/{user_id}/extract-text Extract plain text from a PDF or DOCX (ephemeral)
Request — multipart/form-data
FieldType
filefile (PDF or DOCX)
Response 200
FieldType
textstring (extracted plain text)
Used for job description uploads. The extracted text is returned to the client only — nothing is stored in the database. The user_id path param is for route consistency and is not used for authorisation.
