Per-turn data flow through the backend during an active interview session — from audio and video capture through to Cloud SQL persistence.

sequenceDiagram participant B as Browser participant F as FastAPI participant S as Session Store participant W as Whisper participant CV as CV Pipeline participant G as Gemini participant DB as Cloud SQL Note over B,DB: Session Creation B->>F: POST /sessions F->>S: Create session entry F-->>B: session_id Note over B,DB: Per Turn (repeats for each question) B->>F: POST /sessions/id/next-question F->>G: Conversation history G-->>F: question text F-->>B: question text B->>B: User records audio + video via MediaRecorder B->>F: POST /sessions/id/turns F->>S: Add turn stub F-->>B: turn_id — immediate, non-blocking F-->>W: Background — Transcribe audio via whisper-1 W-->>F: transcript + word-level timestamps F->>S: Update — transcript, filler_words, pause_count, wpm F-->>CV: Background — Sample video frames at 1 per 100ms CV-->>F: Per-frame gaze direction CV-->>F: CVMetrics — gaze_on_camera_pct, gaze_away_events F->>S: Update — confidence_score, confidence_label, observations Note over B,DB: Session End B->>F: POST /sessions/id/end F->>S: Poll until background tasks complete — max 30s F->>S: Aggregate CV scores across all turns F->>G: Prompt with turns, filler counts, pauses, CV summary G-->>F: Score breakdown — dimensions and overall G-->>F: Feedback markdown report F->>DB: Persist — sessions, turns, feedback, cv_analyses F->>S: Delete session from store F-->>B: FeedbackResponse — scores, feedback, confidence_score
Key Properties
PropertyDetail
Turn response latency Immediate — transcription and CV run as BackgroundTasks; the HTTP response is not blocked
Max wait at session end 30 seconds — POST /end polls the store until all pending tasks complete before calling Gemini
Audio persistence Never stored — raw bytes are processed in-memory and discarded after Whisper returns
Video persistence Written to a temp file, processed frame-by-frame, then deleted via os.unlink
Session durability In-memory only during the interview; lost on server restart. 2-hour inactivity TTL with a 5-minute cleanup sweep
DB write strategy Single atomic commit at session end — all tables written together or not at all
Gemini retries Up to 3 attempts if JSON parsing of the response fails; raises RuntimeError after the third failure
