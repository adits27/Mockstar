System Architecture

MockStar

AI-powered mock interview platform — full-stack architecture across browser, backend, external AI services, and persistent storage.

Client
🖥️

Browser (Next.js 16 / React 19)

Single-page application served from Vercel. Handles Google OAuth via NextAuth v5, captures audio and video using the MediaRecorder API, drives the interview state machine, and renders Gemini's markdown feedback.

Next.js 16 React 19 NextAuth v5 Tailwind CSS v4 MediaRecorder API TypeScript
↓ HTTPS · multipart/form-data (audio + video) · JSON
API Layer
⚡

Sessions Router

Create, list, and retrieve completed sessions. Owns the session lifecycle from creation to persistence.

POST /sessions GET /sessions GET /sessions/{id}
🎙️

Turns Router

Accepts audio/video uploads, fires background transcription and CV tasks, drives question generation, and ends the session with aggregated feedback.

POST /turns POST /next-question POST /end
👤

Users Router

Resume upload, retrieval, and document text extraction for job description parsing.

POST /resume GET /resume POST /extract-text
↓ BackgroundTasks · async/await
Service Layer
🎧

Whisper

Transcribes audio with word-level timestamps. Runs as a background task per turn.

whisper-1 verbose_json
📊

Speech Analytics

Detects filler words via regex, counts pauses > 1.5 s from Whisper word gaps, calculates WPM.

analytics.py
👁️

CV Pipeline

Samples 1 frame per 100 ms. Haar cascade for face/eye detection. Gaze classified by iris offset. Outputs confidence score 0–100.

OpenCV cv/analyzer.py
🤖

Gemini

Generates contextual follow-up questions and structured feedback with 5–6 dimension scores.

gemini-2.5-flash feedback.py
↓ asyncpg · SQLAlchemy 2.0 · single commit at session end
State & Storage
⚙️

In-Memory Session Store

Module-level dict in app/store/session_store.py. Holds active sessions during an interview. 2-hour TTL; 5-minute background cleanup sweep. Lost on server restart by design — keeping in-flight state out of the database avoids partial writes.

dict (Python) 2-hour TTL
🗄️

Cloud SQL (PostgreSQL)

Stores completed sessions only. Five tables: sessions, turns, feedback, cv_analyses, user_resumes. Written in a single atomic commit via persist_completed_session.

PostgreSQL asyncpg SQLAlchemy 2.0 Alembic

Data Flows

Browser ↔ API: JSON + multipart
Audio bytes → Whisper (OpenAI API)
Video frames → OpenCV (local)
Prompt → Gemini (Google AI API)
Completed session → Cloud SQL
Active session ↔ In-memory store
